# VAL-007 — Critérios de Decisão da Validação

## 1. Finalidade

Transformar resultados de pesquisa em decisões objetivas, evitando interpretações seletivas ou excessivamente otimistas.

## 2. Estados possíveis

### Go

A proposta apresenta dor relevante, compreensão adequada, valor percebido, confiança suficiente e sinais consistentes de adoção.

### Go com ajustes

A proposta é promissora, mas exige correções específicas de posicionamento, confiança, escopo, recorrência ou segmento prioritário.

### Pivot parcial

A dor existe, mas a solução, narrativa, público ou mecanismo de valor precisa mudar de forma relevante.

### No-Go temporário

As evidências são insuficientes, contraditórias ou fracas. A decisão é interromper expansão e formular novas hipóteses antes de investir mais.

## 3. Critérios mínimos

| Dimensão | Go | Go com ajustes | Pivot parcial / No-Go |
|---|---|---|---|
| Dor | >= 65% moderada/alta | 50% a 64% | < 50% |
| Compreensão correta | >= 80% | 65% a 79% | < 65% |
| Relevância média | >= 8,0 | 7,0 a 7,9 | < 7,0 |
| ITP médio | >= 8,0 | 7,0 a 7,9 | < 7,0 |
| Confiança média | >= 7,0 | 6,0 a 6,9 | < 6,0 |
| Intenção de uso | >= 60% | 45% a 59% | < 45% |
| Recorrência espontânea | >= 50% | 35% a 49% | < 35% |
| Beta | >= 35% | 20% a 34% | < 20% |

## 4. Regras de decisão

- nenhum indicador isolado aprova a proposta;
- compreensão baixa invalida a leitura de interesse alto;
- intenção de uso sem caso real de recorrência deve ser tratada com cautela;
- disposição para pagar não é obrigatória para validar o problema inicial;
- segmentos com resultados muito diferentes devem ser avaliados separadamente;
- evidência comportamental futura pode superar intenção declarada;
- uma decisão Go não autoriza construir todo o ecossistema de uma vez.

## 5. Critérios de ajuste prioritário

- compreensão baixa: revisar narrativa;
- dor baixa: revisar público ou problema;
- confiança baixa: revisar privacidade, controle e posicionamento;
- valor alto e recorrência baixa: revisar proposta diária e casos de uso;
- beta alto e pagamento baixo: validar primeiro valor gratuito e modelo econômico depois;
- polarização alta: identificar segmento de maior aderência.

## 6. Registro da decisão

Cada rodada deverá registrar:

- decisão;
- evidências favoráveis;
- evidências contrárias;
- limitações;
- segmentos prioritários;
- mudanças aprovadas;
- hipóteses para nova rodada;
- responsável e data.
